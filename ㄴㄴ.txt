[Dynamic Web Project 생성]
Default output folder : 저장할때마다 .class 파일이 저장되는데 그 파일이 저장되는 위치 (java project 의 경우 bin 이라는 이름의 폴더 안에 저장)
Context root: Web서버의 프로그램이 인식하는 웹싸이트가 여러개 등록되어있다면, 그것들을 구분하는 이름. 프로젝트 이름이랑 굳이 똑같지는 않아도됨. 변경가능


[Web Modules 설정]
Path : 논리적인 URL 에서 인정되는 경로
Document Base : 실제 파일이 저장된 곳. 
브라우저에서 Path에 작성해놓은 경로를 치면 Document Base에 작성된 프로젝트 안에서 파일을 찾는다.

* Modules 설정 변경하면, 서버를 재실행 시켜줘야한다.


** html 파일명의 대소문자만 바꾸는 경우
톰캣의 배포과정에서는 대소문자 구분을 하지않기 때문에 단순히 대소문자만 바꿔서 저장하면 인식을 못하는 경우 있음.(파일을 못찾음)
배포할때 이미 파일이  존재한다고 인지해서 바뀐 파일명에 대한 파일을 처리해주지 않음.
하지만 인식과정에서는 대소문자 구분을 해주기 때문에 당연히 해당 파일이 존재하지 않는다고 처리됨.
-> 해결: 다른 문자 추가해서 바꿨다가 다시 대소문자 바꾼 파일명으로 2번에 걸친 과정으로 바꿔줘야함
